структура данных для обновления БД
0. пользователи
{
	0.0 добавление нового пользователя
		- идентификатор пользователя (нужен на сайте для авторизации, уникален)
		- фамилия
		- имя
		- отчество
		- тип пользователя (клиент, торговый представитель, диспетчер, администратор)
		- зашифрованный пароль
		- телефон
		- эл. почта
		- идентификатор пользователя, который инициировал добавление записи

	0.1 изменение данных о пользователе
		- идентификатор пользователя
		- идентификатор пользователя, который инициировал изменение записи
		(далее нужно отправить только те данные, которые нуждаются в изменении)
		- фамилия
		- имя
		- отчество
		- тип пользователя (клиент, торговый представитель, диспетчер, администратор)
		- зашифрованный пароль
		- телефон
		- эл. почта

пример запроса на добавление/изменение:
{"userIdentificator":"100000099","firstName":"Имя","lastName":"Фамилия","patronymic":"Отчество","UserType":"CLIENT","pass":"password","telephone":"+79201690412","email":"login@snail.con","lastModBy":"100000999"}

UserType - перечисляемый тип, имеющий 4 значения:
	CLIENT - клиент
	ADMIN  - администратор
	MARKET - торговый представитель
	DISPATCHER - диспетчер
	W_DISPATCHER - диспетчер склада

	0.2 временное удаление пользователя
		- идентификатор временно удаляемого пользователя
		- идентификатор пользователя, который инициировал изменение записи
	0.3 восстановление пользователя
		- идентификатор временно удаленного пользователя
		- идентификатор пользователя, который инициировал изменение записи
	0.4 удаление пользователя
		- идентификатор удаляемого пользователя
		- идентификатор пользователя, который инициировал изменение записи
		
пример запроса на удаление:
{"userIdentificator":"100000099","action":"REMOVE","lastModBy":"100000999"}

action - перечисляемый тип, имеющий 3 значения:
	REMOVE - временно удалить
	ACTIVE - восстановить
	DELETE - удалить насовсем
	
итоговое примерное содержимое файла:
[{"userIdentificator":"100000099","firstName":"Имя","lastName":"Фамилия","patronymic":"Отчество","UserType":"CLIENT","pass":"password","telephone":"+79201690412","email":"login@snail.con","lastModBy":"100000999"},{"firstName":"Алексей","lastName":"Баронов","patronymic":"Валентинович","userIdentificator":"100000099","lastModBy":"100000999"},{"userIdentificator":"100000099","action":"REMOVE","lastModBy":"100000999"},{"userIdentificator":"100000099","action":"REPAIR","lastModBy":"100000999"},{"userIdentificator":"100000099","pass":"1234567890","lastModBy":"100000999"},{"userIdentificator":"100000099","action":"DELETE","lastModBy":"100000999"}]
}
1. пункты
{
	1.0 добавление нового пункта:
		- имя пункта
		- идентификатор пункта (номер уникален, нужен для поиска пунктов)
		- тип пункта (клиент, торговое представительство, склад)
		- для типа пункта "Клиент" идентификатор объединения пунктов назначения
		- адрес:
		  - регион
		  - район
		  - нас. пункт
		  - адрес (улица, дом)
		- эл. почта
		- телефон
		- ответственные (список людей, кто сможет проставлять статусы для накладных)
		- идентификатор пользователя, который инициировал добавление записи
	1.1 изменение данных
		- идентификатор пункта
		- идентификатор пользователя, который инициировал изменение записи
		(далее нужно отправить только те данные, которые нуждаются в изменении)
		- имя пункта
		- тип пункта (клиент, торговый представитель, склад)
		- для типа пункта "Клиент" идентификатор объединения пунктов назначения
		- адрес:
		  - регион
		  - район
		  - нас. пункт
		  - адрес (улица, дом)
		- эл. почта
		- телефон
		- ответственные (список людей, кто сможет проставлять статусы для накладных)
		
пример запроса на добавление/изменение:
{"pointName":"name","pointIdentificator":"110000999","pointType":"CLIENT","region":"Тверская область","district":"","locality":"Тверь","address":"улица Мичурина, д.19","email":"fbi@usa.gov","telephone":"+74822522600","users":["100000099", "100000999"],"lastModBy":"100000999"}
pointType - перечисляемый тип, имеющий 3 значения:
	- CLIENT - адрес клиента
	- MARKET - адрес торгового представителя
	- WAREHOUSE - склад
	
	1.2 удаление пункта
	- номер удаляемого пункта
	- идентификатор пользователя, который инициировал изменение записи

пример запроса на удаление:
{"pointIdentificator":"110000999","lastModBy":"100000999"}

2. документы (накладные, вн. заявки, заявки - данные выгружаются для каждой внутренней заявки в 1С, кроме вн. заявок в статусе "Не определена")

		- идентификатор пользователя, который инициировал создание записи

		- идентификатор пользователя, который инициировал изменение записи
		(далее нужно отправить только те данные, которые нуждаются в изменении)
		- идентификатор (номер) заявки
		- идентификатор (номер) внутренней заявки
		- дата и время создания
		- идентификатор торгового представителя
		- идентификатор клиента
		- идентификатор склада (пункт)
		- количество коробок

3. маршруты (в том числе и маршруты для маршрутных листов) (представляет из себя список пунков, являющихся узлами маршрута и расставленными в порядке посещения)
{

		- идентификатор маршрута
		- идентификаторов пунктов
		- номер пункта по порядку маршрута

		- время ПРР в пункте
		- время до следующего пункта
		- расстояние до следующего пункта
		- идентификатор пользователя, который инициировал создание записи
	3.1 изменение маршрута
		- идентификатор маршрута
		- идентификатор пользователя, который инициировал изменение записи

		- номер пункта по порядку маршрута

		- время ПРР в пункте
		- время до следующего пункта
		- расстояние до следующего пункта

		- данные о водителе
		- список номеров накладных для данного маршрутного листа
